## Spacing Scale

All tokens carry forward from Phase 1 unchanged. Used in Phase 2 as follows:

| Token | Value | CSS Custom Property | Phase 2 Usage |
|-------|-------|---------------------|---------------|
| xs | 4px | `--space-xs` | Gap between project title label and project type label |
| sm | 8px | `--space-sm` | Grid gap between home page image cells (see exception below) |
| md | 16px | `--space-md` | Page horizontal margin (inner), text section padding between columns |
| lg | 24px | `--space-lg` | Section vertical gap (cover-to-text-section, text-section-to-gallery) |
| xl | 32px | `--space-xl` | Page horizontal outer margin on desktop, gallery image vertical gap |
| 2xl | 48px | `--space-2xl` | Vertical padding above/below the 2-column text section |
| 3xl | 64px | `--space-3xl` | Page-level top padding (below nav), footer top padding |

**Exception — home page grid gap:** Grid `gap` is set to `var(--space-sm)` (8px). This resolves the "8–16px" range in CONTEXT.md D-04 to 8px — consistent with the tightest editorial feel and the grid's high image density.

**Exception — no additional component tokens added.** `--nav-height` and `--touch-target` carry forward from Phase 1 without change.

**Source:** Phase 1 UI-SPEC spacing scale; CONTEXT.md D-04 (gap range resolved to 8px by Claude's discretion)

## Typography

All tokens carry forward from Phase 1. Phase 2 adds Bold (700) weight for project page headings.

| Role | Size | Weight | CSS Weight | Line Height | Usage in Phase 2 |
|------|------|--------|------------|-------------|------------------|
| Body | 16px `--font-size-base` | Regular | 400 | 1.5 | Project description, bio text, footer text |
| Label | 14px `--font-size-sm` | Regular | 400 | 1.4 | Project type label, "View project" link, social links, prev/next link text |
| Heading | 24px `--font-size-lg` | Bold | 700 | 1.2 | Project page title (h1), about page section heading |
| Display | 40px `--font-size-xl` | Bold | 700 | 1.1 | Reserved — not used in Phase 2 |

**Two weights only:** `--font-weight-regular` (400) and `--font-weight-bold` (700). No medium weight is used in Phase 2. All label-level elements (project type, "View project" link, prev/next link text, social links) use Regular (400) to preserve clear Bold/Regular contrast for headings vs. body and labels.

**Bold weight addition for Phase 2:**

Add to `src/styles/global.css`:
```css
@font-face {
  font-family: 'Neue Haas Unica';
  src: url('/fonts/NeueHaasUnica-Bold.woff2') format('woff2');
  font-weight: 700;
  font-style: normal;
  font-display: swap;
}
```

Add to `src/styles/tokens.css`:
```css
--font-weight-bold: 700;
```

**Project metadata 2-column text section typography:**
- Left column — project name: `--font-size-lg` (24px), `--font-weight-bold` (700), `--line-height-heading` (1.2)
- Left column — project type: `--font-size-sm` (14px), `--font-weight-regular` (400), `--color-text-muted`, margin-top `--space-xs` (4px)
- Right column — description: `--font-size-base` (16px), `--font-weight-regular` (400), `--line-height-body` (1.5)
- Right column — live demo link: `--font-size-sm` (14px), `--font-weight-regular` (400), `--color-text`, underline on hover

**Source:** Phase 1 UI-SPEC (typography table carry-forward); Phase 1 CONTEXT.md D-02 (Bold deferred to Phase 2); CONTEXT.md D-08 (2-column text section); Claude's discretion (exact size assignments for metadata columns); revision fix (medium weight removed, reassigned to regular)

## Color

All tokens carry forward from Phase 1 unchanged.

| Role | Value | CSS Custom Property | Phase 2 Usage |
|------|-------|---------------------|---------------|
| Dominant (60%) | `#FFFFFF` | `--color-bg` | Page canvas, home grid background, project page background, about page background |
| Secondary (30%) | `#F5F5F5` | `--color-surface` | Footer background — subtle surface separation from page body |
| Accent (10%) | `#111111` | `--color-accent` | Prev/next navigation link underline on hover only |
| Text | `#111111` | `--color-text` | All body copy, project titles, bio text, social links, footer text |
| Text muted | `#999999` | `--color-text-muted` | Project type label, caption text |
| Destructive | `#CC0000` | `--color-destructive` | Not used in Phase 2 |

**60/30/10 breakdown:**
- 60% — `#FFFFFF` white: all page backgrounds, full canvas dominance
- 30% — `#F5F5F5` off-white: footer strip only
- 10% — `#111111` near-black: accent hover interaction on prev/next links; all text is `--color-text` which happens to share the same hex but is semantically distinct

**Accent reserved for:** prev/next navigation link hover underline only. No other Phase 2 elements receive the accent treatment.

**Source:** Phase 1 UI-SPEC (color table carry-forward); Claude's discretion (footer surface color, accent assignment)

## Component Inventory

### Home Page Grid (`src/pages/index.astro`)

**Focal point:** The cover image grid is the sole visual anchor — no competing elements on the page.

**Structure:**
```
<main>
  <ul class="project-grid">
    <li class="project-cell">
      <a href="/projects/{slug}">
        <img src="{cover}" alt="{title}" loading="lazy" />
      </a>
    </li>
    …
  </ul>
</main>
```

**Visual spec:**
- Layout: CSS Grid, `grid-template-columns: repeat(2, 1fr)` on desktop; `1fr` on mobile
- Grid gap: `var(--space-sm)` (8px) in both axes
- Page margin: `padding-inline: var(--space-xl)` (32px) on desktop; `padding-inline: var(--space-md)` (16px) on mobile
- Image aspect ratio: `aspect-ratio: 3 / 2` enforced via CSS on each `<img>`; `object-fit: cover`
- Images: lazy-loaded (`loading="lazy"`), served as WebP with responsive srcset via Astro `<Image>` component
- No titles, no overlays, no hover text — images only (CONTEXT.md D-02)
- Hover: no visual state change on image cells — editorial restraint per aesthetic
- Top padding: `padding-top: var(--space-3xl)` (64px) below the nav

**Mobile breakpoint:** Single column below `640px`. Grid collapses to `grid-template-columns: 1fr`.

**Source:** CONTEXT.md D-01, D-02, D-03, D-04, D-05; REQUIREMENTS.md HOME-01 through HOME-05; Claude's discretion (hover behavior, exact padding values)

### Project Detail Page (`src/pages/projects/[slug].astro`)

**Structure:**
```
<main>
  <!-- 1. Full-width cover image -->
  <div class="project-cover">
    <img src="{cover}" alt="{title}" />
  </div>

  <!-- 2. 2-column metadata section -->
  <section class="project-meta">
    <div class="project-meta__left">
      <h1>{title}</h1>
      <p class="project-type">{type}</p>
    </div>
    <div class="project-meta__right">
      <p>{description}</p>
      <!-- conditional -->
      <a href="{liveUrl}" class="project-demo-link" target="_blank" rel="noopener noreferrer">
        View project
      </a>
    </div>
  </section>

  <!-- 3. Gallery images stacked vertically -->
  <div class="project-gallery">
    <img src="{image}" alt="" /> <!-- one per row -->
    …
  </div>

  <!-- 4. Prev/next navigation -->
  <nav class="project-nav" aria-label="Project navigation">
    <a href="/projects/{prevSlug}" class="project-nav__prev">{prevTitle}</a>
    <a href="/projects/{nextSlug}" class="project-nav__next">{nextTitle}</a>
  </nav>
</main>
```

**Visual spec — Cover image:**
- Width: 100% (full content width, within page margins)
- Height: `50vh` with `object-fit: cover` — gives consistent cinematic strip without overriding page margins
- No text overlay (CONTEXT.md D-07)
- Page margins: `margin-inline: var(--space-xl)` (32px desktop), `var(--space-md)` (16px mobile)

**Visual spec — 2-column metadata section:**
- Layout: CSS Grid `grid-template-columns: 1fr 1fr` on desktop; single column on mobile (left column stacks above right column)
- Top/bottom padding: `var(--space-2xl)` (48px) above and below the section
- Column gap: `var(--space-xl)` (32px)
- Left column: project name (`--font-size-lg`, bold, `--line-height-heading`), project type below (`--font-size-sm`, regular, `--color-text-muted`, `margin-top: var(--space-xs)`)
- Right column: description (`--font-size-base`, regular, `--line-height-body`), optional live demo link below with `margin-top: var(--space-md)` (16px)
- Live demo link label: "View project" — underline decoration, no icon
- Live demo link hidden entirely if `liveUrl` is absent (REQUIREMENTS.md PROJ-05)

**Visual spec — Gallery:**
- Each image: full content width, `margin-inline: var(--space-xl)` (32px desktop)
- Vertical gap between images: `var(--space-xl)` (32px)
- Top gap from metadata section to first image: `var(--space-lg)` (24px)
- Images: lazy-loaded, WebP, responsive srcset via Astro `<Image>` component
- No captions

**Visual spec — Prev/next navigation:**
- Layout: flexbox row, `justify-content: space-between`
- Padding: `var(--space-2xl)` (48px) top and bottom, `var(--space-xl)` (32px) inline
- Font: `--font-size-sm` (14px), `--font-weight-regular` (400), `--color-text`
- Hover: text underline `2px solid var(--color-accent)` below the project name
- Prev link left-aligned; Next link right-aligned
- If no previous project: prev slot is empty; if no next project: next slot is empty

**Mobile collapse for metadata 2-column:** Single column below `640px`. Left column (title + type) stacks on top of right column (description + link). Full-width.

**Source:** CONTEXT.md D-07, D-08, D-09, D-10; REQUIREMENTS.md PROJ-01 through PROJ-05; Claude's discretion (cover height 50vh, metadata column gap, prev/next styling, mobile collapse rule)

- Link labels: "LinkedIn" and "Instagram" — plain text, no icons, no arrow glyphs
- No CTA, no contact form (REQUIREMENTS.md ABOUT-04)

## Interaction Contract

### Home Page — Image Grid

| Trigger | Behavior |
|---------|----------|
| Click/tap any image cell | Navigate to `/projects/{slug}` |
| Hover | No visual state change — editorial restraint |
| Image load | Lazy-loaded; placeholder is the `--color-surface` background color of the cell |

### Project Page — Live Demo Link

| Trigger | Behavior |
|---------|----------|
| Click "View project" | Opens external URL in new tab (`target="_blank" rel="noopener noreferrer"`) |
| Link absent (no `liveUrl`) | Element not rendered — no empty placeholder shown |

### Project Page — Prev/Next Navigation

| Trigger | Behavior |
|---------|----------|
| Click prev/next project name | Navigate to adjacent project page |
| Hover | Underline appears: `text-decoration: underline` |
| No adjacent project | Corresponding link is absent from the DOM — no disabled state |

### About Page — Social Links

| Trigger | Behavior |
|---------|----------|
| Click LinkedIn or Instagram | Opens in new tab (`target="_blank" rel="noopener noreferrer"`) |
| Hover | Underline appears: `text-decoration: underline` |

**No animations beyond nav hide/show behavior** — per REQUIREMENTS.md DSGN-04. All hover states use instant opacity or underline changes only (no transitions).
