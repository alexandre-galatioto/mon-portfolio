  <action>
Add a `<link rel="preload">` tag for the Regular (400) font weight in the `<head>` section of `src/layouts/BaseLayout.astro`. Insert it after the viewport meta tag and before the `<title>` tag.

The target `<head>` block becomes:

```html
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <meta name="description" content={description} />
  <link
    rel="preload"
    href="/fonts/NeueHaasUnica-Regular.woff2"
    as="font"
    type="font/woff2"
    crossorigin
  />
  <title>{title}</title>
</head>
```

Critical details:
- The `crossorigin` attribute is MANDATORY even for same-origin fonts — omitting it causes the browser to fetch the font twice (per D-06, RESEARCH.md Pattern 5).
- Font file path is `/fonts/NeueHaasUnica-Regular.woff2` (confirmed at `public/fonts/NeueHaasUnica-Regular.woff2`).
- Do NOT add preload for Medium (500), Bold (700), or Italic (400i) weights — those load on demand per D-06.
- Do NOT change any other part of BaseLayout.astro — only insert the single `<link>` tag.
