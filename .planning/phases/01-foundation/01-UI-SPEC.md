## Spacing Scale

Declared values (multiples of 4):

| Token | Value | CSS Custom Property | Usage |
|-------|-------|---------------------|-------|
| xs | 4px | `--space-xs` | Icon gaps, inline element padding |
| sm | 8px | `--space-sm` | Compact element spacing, nav link gap |
| md | 16px | `--space-md` | Default element spacing |
| lg | 24px | `--space-lg` | Nav horizontal padding, section padding |
| xl | 32px | `--space-xl` | Layout section gaps |
| 2xl | 48px | `--space-2xl` | Major section vertical breaks |
| 3xl | 64px | `--space-3xl` | Page-level top/bottom padding |

**Nav-specific:**

| Token | Value | CSS Custom Property | Usage |
|-------|-------|---------------------|-------|
| nav-height | 64px | `--nav-height` | Navigation bar height (also defines layout top offset) |
| touch-target | 48px | `--touch-target` | Minimum hit area for hamburger button (mobile); 48px exceeds WCAG 2.5.5 minimum of 44px and aligns to the standard spacing set |

Exceptions: `--nav-height` (64px) is a non-standard value for a specific component need. `--touch-target` (48px) maps to `--space-2xl` in the standard scale.

**Source:** Discretion (CONTEXT.md Claude's Discretion — spacing scale and nav height are explicitly delegated)

## Component Inventory

### Nav Component (`src/components/Nav.astro`)

**Visual hierarchy:** The nav logo/name is the primary focal point; nav links (Work, About) are secondary and visually subordinate in weight and scale.

**Structure:**
```
<nav>
  <a class="nav-logo">Name/Logo</a>          ← left
  <ul class="nav-links">
    <li><a>Work</a></li>
    <li><a>About</a></li>
  </ul>                                       ← right, desktop only
  <button class="nav-hamburger">             ← right, mobile only
    <span class="bar"></span> × 3
  </button>
</nav>
<div class="nav-overlay">                    ← full-screen mobile overlay
  <ul>Work / About links centered</ul>
  <button class="nav-close">X</button>
</div>
```

**Visual states:**

| State | Description |
|-------|-------------|
| Default (visible) | Full-width bar, `--nav-height: 64px`, white background, `position: sticky; top: 0` |
| Hidden (scroll down) | `transform: translateY(-100%)` applied after 80px scroll threshold |
| Revealed (scroll up) | `transform: translateY(0)` with `transition: transform 200ms ease` |
| Mobile hamburger | Hamburger button visible, nav links hidden; 3 horizontal lines, 2px height, 24px width, `--space-sm` gap between lines |
| Mobile overlay open | Full-viewport overlay, `background: #FFFFFF`, links stacked and centered, 24px font, weight 500 |
| Nav link active | Bottom border `2px solid var(--color-accent)`, offset `2px` below baseline |
| Nav link hover | Opacity 0.6, no underline change |

**Scroll behavior implementation:**
- Inline `<script>` block inside `Nav.astro` (not a bundled JS file — compliant with PERF-02)
- Scroll threshold: 80px before hide triggers
- Direction detection: compare `window.scrollY` to previous stored value on `scroll` event

**Source:** CONTEXT.md D-12, D-13, D-14, D-15, D-16; Discretion (scroll threshold 80px, transition timing, overlay styles)

## Interaction Contract

### Scroll Hide/Show (NAV-02)

| Trigger | Behavior |
|---------|----------|
| Scroll down past 80px | Nav translates off-screen: `transform: translateY(calc(-1 * var(--nav-height)))` |
| Scroll up (any distance) | Nav translates back: `transform: translateY(0)` |
| Transition | `transition: transform 200ms ease` on the `<nav>` element |
| At page top (scrollY = 0) | Nav always visible regardless of direction |

### Mobile Hamburger (NAV-03)

| Trigger | Behavior |
|---------|----------|
| Tap hamburger button | Overlay appears: `opacity: 1; visibility: visible` transition 150ms |
| Tap close button or overlay link | Overlay disappears: `opacity: 0; visibility: hidden` transition 150ms |
| Body scroll while overlay open | `overflow: hidden` on `<body>` to prevent scroll-behind |

**No decorative animations beyond the above** — per REQUIREMENTS.md DSGN-04.
